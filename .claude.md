# vLLM Study Preferences

- Write study articles as HTML, preferably as standalone pages that can be opened directly under `site/`.
- The final article content should be written in Chinese.
- Prioritize code reading and code walkthroughs. Ground explanations in concrete files, functions, important snippets, and execution paths.
- Use diagrams to improve understanding. Each article should preferably include an overall architecture diagram, module relationship diagram, or data-flow diagram.
- When tensor transformations, parallel sharding, communication, cache layout, or kernel inputs/outputs are involved, include shapes and explain what each dimension means.
- For complex flows, include concrete examples such as token counts, patch counts, rank assignments, shape changes, or input/output data flow.
- Prefer the narrative order: how to use it, overall architecture, then implementation details and code walkthrough.

## Commit And Push Habit

- After finishing an article or repository change, commit the intended files and push directly to `main`.
- Do not create a separate pull request.
- Push to the user's configured remote repository after committing.
- Do not include unrelated local files such as `.DS_Store` in commits.
