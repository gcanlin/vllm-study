Qwen3 模型结构图解 — vLLM 视角

配合源码: vllm/model_executor/models/qwen3.py、qwen2.py、layers/linear.py。 读完后你应该能回答: ① Qwen3 一层 forward 里张量 shape 怎么变？ ② QKV/Row/Column Parallel 在几何上各切了哪个轴？ ③ qkv.split([...], dim=-1) 到底在干嘛？

本文使用的"具体数字"约定（以 Qwen3-8B 为例）：

注意 n_h·d = 32·128 = 4096 = H，而 n_kv·d = 8·128 = 1024 ≠ H。这就是 GQA：K/V 的"宽度"比 Q 窄 4 倍。

0. 顶层骨架

整个模型像一个三明治：

对应代码（Qwen3ForCausalLM）：

self.model = Qwen3Model(...)       # embed + L 层 + final norm
self.lm_head = ParallelLMHead(V, H) # 投影回词表 (或与 embed_tokens 权重共享)

真正复杂的是中间那个 L × Qwen3DecoderLayer。下面把它一层剖开。

1. 一个 DecoderLayer 的几何

对照 Qwen3DecoderLayer.forward，一个 token 进入一层时走的路：

两条虚线橙色就是残差（residual）。vLLM 在 RMSNorm 里做了一个小技巧：把"上一层的残差 + 当前归一化"融合成一次调用：

hidden_states, residual = self.input_layernorm(hidden_states, residual)
# 等价于:
#   hidden_states = hidden_states + residual         (上一层延迟的 add)
#   residual      = hidden_states                    (准备好新的残差)
#   hidden_states = rmsnorm(hidden_states)
所以 forward 末尾返回的 (hidden_states, residual) 并不是普通残差和——它是"还没相加"的状态，下一层入口才合并。这是优化算子能省一次显存往返。

2. Self-Attention 内部的 8 个 shape 转换

这是 Qwen3Attention.forward 的全部 8 行代码（剥掉条件分支）：

qkv, _ = self.qkv_proj(hidden_states)                          # ①
q, k, v = qkv.split([q_size, kv_size, kv_size], dim=-1)         # ②
q_by_head = q.view(*q.shape[:-1], -1, head_dim)                 # ③ q-norm reshape
q = self.q_norm(q_by_head).view(q.shape)                        # ④ q-norm + 还原
k_by_head = k.view(*k.shape[:-1], -1, head_dim)                 # ③' k-norm reshape
k = self.k_norm(k_by_head).view(k.shape)                        # ④' k-norm + 还原
q, k = self.rotary_emb(positions, q, k)                         # ⑤ RoPE
attn_output = self.attn(q, k, v)                                # ⑥ flash/paged attention
output, _ = self.o_proj(attn_output)                            # ⑦
return output                                                   # ⑧

下面逐步把每个 shape 画出来（先假设 TP=1，纯单卡）：

① qkv_proj : [B, S, H] → [B, S, q_size + kv_size + kv_size]

"为什么不分三个 Linear 而是合一起？" —— GEMM 越大越快、kernel launch 越少。所以 vLLM 把 Q、K、V 三个权重沿输出维度 concat在一起，存成一个矩阵 W_qkv：

Q-part K-part V-part

具体宽度（TP=1，Qwen3-8B）：

  • q_size = n_h · d = 32·128 = 4096
  • kv_size = n_kv · d = 8·128 = 1024
  • 合起来输出宽度 = 4096 + 1024 + 1024 = 6144

② qkv.split([q_size, kv_size, kv_size], dim=-1)

Tensor.split(sizes, dim) 语法解释：
  • 第一个参数如果是 list / tuple，表示"按这些长度依次切片"，长度和必须等于该维度的大小；
  • 第一个参数如果是 int，表示"每段长度都是这个 int"；
  • 返回一个 tuple，元素数 = 切片数；
  • 这是 view 操作，不复制内存——只是建了 3 个指向原 buffer 不同 offset 的视图。

所以这一行做的事，就是把上面那条 [B,S,6144] 的拼接张量"按 [4096, 1024, 1024]"切回 3 段：

③④ q-norm / k-norm：reshape 出"头"这个维度

Qwen3 的一个新东西：对 Q 和 K 在每个 head 内部单独做一次 RMSNorm（QK-Norm）。RMSNorm 是沿最后一维做的，所以我们必须先把"flat 的 q"reshape 成"按 head 切开"：

q : [B, S, n_h·d] ─view→ q_by_head : [B, S, n_h, d]

所以代码中：

q_by_head = q.view(*q.shape[:-1], q.shape[-1] // self.head_dim, self.head_dim)
# -1 写法等价 → q.view(B, S, n_h_local, d)

之后 self.q_norm 在最后一维 d 上做 RMS，再 view 回 [B, S, n_h·d]。

⑤ RoPE — 旋转位置编码

对 Q/K 的每个 head 中相邻 2 个维度做"按 position 转一定角度"的旋转。它不改变 shape，只是在 q、k 上原地修改数值，让点积自带相对位置信息。V 不参与。

⑥ self.attn(q, k, v) — vLLM 的 PagedAttention

这一步内部含义就是经典的：

attn = softmax( Q · Kᵀ / √d ) · V

但在 vLLM 里：K、V 不是当前 batch 自己的，而是从 KV cache 里取（含历史 token）。最后输出形状还是 [B, S, n_h·d] = [B, S, H]。

⑦ o_proj — 把 attention 输出投回 H 维

attn_output [B,S,H] · W_o [H, H] → output [B,S,H]

o_proj 是 RowParallelLinear，原因下面 §4 解释。

3. MLP（SwiGLU）

Qwen3 用的是 SwiGLU：

MLP(x) = ( SiLU(x · W_gate) ⊙ (x · W_up) ) · W_down

vLLM 把 W_gate 和 W_up 合并成一个 gate_up_proj（与 QKV 同样思路：合并 GEMM）：

gate up

SiluAndMul 是 vLLM 的融合算子，等价于：

gate, up = gate_up.chunk(2, dim=-1)   # 概念上
y = silu(gate) * up                   # 逐元素

之后 down_proj（RowParallelLinear）把 F 维投回 H：

y [B,S,F] · W_down [F, H] → out [B,S,H]

4. Tensor Parallel：三种切法的几何

这是你最想搞清楚的部分。先记两条主线：

Column Parallel: 把权重按"输出维度"切，每张 GPU 负责一段输出列；输入复制到每张卡；输出在卡之间是拼接关系。
Row Parallel: 把权重按"输入维度"切，每张 GPU 负责一段输入行；输入已经被切过；输出在卡之间是逐元素求和关系（需要 all-reduce）。

4.1 Column Parallel（=QKVParallelLinear, MergedColumnParallelLinear）

数学上 y = x · W，把 W 横向劈成 W = [W₁ | W₂]，则：

x · [W₁ | W₂] = [x·W₁ | x·W₂]

所以两张 GPU 各自算 y_i = x · W_i，不需要任何通信就拿到了"输出的一段"。

QKVParallelLinear 的"沿 head 切"是什么意思？

QKVParallelLinear 在逻辑上是 Column Parallel，但它按 head 数来切而不是按 q_size 的元素数。为什么？因为每个 head 是一个独立的 attention 计算单元，你不能把同一个 head 的 d 维切到两张卡上——那样 softmax 都做不了。

看代码（linear.py 中 QKVParallelLinear.__init__）：

self.num_heads    = total_num_heads    // tp_size   # Q 头数 / 卡
self.num_kv_heads = max(1, total_num_kv_heads // tp_size)
# Q 的 output 宽度 / 卡 = num_heads * head_size
# K 的 output 宽度 / 卡 = num_kv_heads * head_size
# V 的 output 宽度 / 卡 = num_kv_heads * v_head_size

以 Qwen3-8B + TP=2 为例：

TP=1TP=2 / 单卡
Q heads3216
KV heads84
q_size (单卡)40962048
kv_size (单卡)1024512
qkv 输出宽度 (单卡)61443072

所以每张卡的 qkv_proj 物理上输出 [B,S,3072]，分别切成 q[B,S,2048] / k[B,S,512] / v[B,S,512]，用同一行 split 代码，因为里面引用的 self.q_size/self.kv_size 是"已经按 TP 切过的本地大小"。

几何直观：32 个 Q 头被横着掰成两半，每张卡拿 16 个头独立算 attention。Attention 输出也只有 16 个头的宽度。直到 o_proj 才把"分散到各卡上的头"合并回完整的 H 维表示——这就是为什么 o_proj 是 Row Parallel。

n_kv < tp_size 的特殊情况：KV 头被复制

例如 n_kv=8，TP=16。这时 num_kv_heads = max(1, 8//16) = 1，num_kv_head_replicas = 16/8 = 2：每个 KV head 被 2 张卡共享拷贝一份。Q 头总数仍是 n_h // tp 切下去。

4.2 Row Parallel（=RowParallelLinear, o_proj, down_proj）

数学上把 W 纵向劈成两段：

x · W = [x₁ | x₂] · [ W₁
W₂ ] = x₁·W₁ + x₂·W₂

即把输入按通道切两半，每张卡做一个"窄"的 GEMM，最后用 all-reduce 加起来。

vLLM 实现里这步通信由 tensor_model_parallel_all_reduce(output_parallel) 完成（RowParallelLinear.forward 里）。

4.3 Column → Row 的"夹心结构"

vLLM 故意让每个块（Attention、MLP）都是 Column → Row 这种"夹心三明治"：

MLP 是一样的套路：

x (完整) ─Col→ gate_up_proj ─→ [B,S, 2F]/卡 ─→ SiluAndMul ─→ [B,S, F]/卡 ─Row→ down_proj + all-reduce ─→ x (完整)
每一层 forward 中 只有 2 次 all-reduce（attention 出口一次、MLP 出口一次），这是 Megatron-LM 经典张量并行设计。整个 L 层模型一共 2L 次跨卡通信。

6. 收尾：lm_head 与采样

L 层都跑完后：

final_norm  → [B, S, H]
lm_head     → [B, S, V]      # ParallelLMHead，沿 V 切分
logits_proc → 采样 / top-k / top-p

ParallelLMHead 也是 column parallel（按 vocab 维切），最后聚合或者直接交给 sampler 各卡分别采各自的"vocab 子段"。

7. 一句话总结

  1. QKVParallelLinear = Column Parallel + 按 head 切，把 3 个矩阵合一个 GEMM。
  2. qkv.split([q,kv,kv], -1) = 零拷贝的视图切分，把那个合并 GEMM 的结果拆回 Q/K/V。
  3. RowParallelLinear = 按输入维度切的 GEMM，结尾必须做 all-reduce。
  4. 每个块都是"Col 进 → 本地计算 → Row 出"夹心，全模型每层只 2 次 all-reduce。
  5. 残差在 vLLM 里被 RMSNorm 算子吃进去做 fusion，所以你看到 (hidden_states, residual) 这种成对返回。

读到这里再回去看 qwen3.py，你应该觉得每一行都明明白白了。
