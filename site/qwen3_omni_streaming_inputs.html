1. 三种输入先分清

输入类型当前代码里的入口是不是 engine 级 streaming inputQwen3-Omni 里发生什么
音频 realtime /v1/realtime，事件 input_audio_buffer.append / commit 是。音频 chunk 先进入 queue，再被 buffer_realtime_audio 切成多次 StreamingInput。 每个音频 segment 变成带 multi_modal_data={"audio": segment} 的 TokensPrompt。
文本输入 /v1/chat/completions 或 video stream 里的 video.query.text 通常不是。文本是一次 prompt 的一部分；流式主要体现在输出 delta。 文本被 renderer/chat template 转成 token ids，进入 Thinker。video stream 里文本 query 会和当前帧/音频 buffer 拼成一次 chat request。
视频输入 /v1/video/chat/stream，事件 video.frame / video.query 不是逐帧进入 engine。WebSocket 层持续收帧；每次 query 把当前 frame buffer 采样成一次 prompt。 帧被转成 image_pil 或 image_url content block；Qwen3-Omni processor 把它当 image/video 视觉 token 处理。 
/v1/realtime audio
  websocket audio append -> audio_queue -> buffer_realtime_audio()
  -> AsyncGenerator[StreamingInput] -> AsyncOmni add_request / streaming_update
  -> Qwen3 Thinker -> Talker -> Code2Wav -> response.audio.delta

/v1/video/chat/stream
  websocket video.frame/audio.chunk -> session buffers
  video.query(text) -> sample frames + current audio + text -> ChatCompletionRequest
  -> engine.generate once per query -> response.text.delta + optional response.audio.delta

/v1/chat/completions
  messages text/audio_url/video_url -> renderer + OmniInputPreprocessor
  -> Qwen3 Thinker multimodal embeddings -> output stream if stream=true
所以，“音频 realtime”是 engine 层真正的 streaming input；“视频 stream”是协议层持续收集视频帧，query 时快照成一次多模态请求；“文本输入”目前主要是 request/query 边界的输入，文本 streaming 更多体现在输出 delta。

2. 音频 realtime streaming

上游 vLLM 类 RealtimeConnection，方法 handle_event

async def handle_event(self, event: dict):
    """Route events to handlers.

    Supported event types:
    - session.update: Configure model
    - input_audio_buffer.append: Add audio chunk to queue
    - input_audio_buffer.commit: Start transcription generation
    """
    event_type = event.get("type")
    if event_type == "session.update":
        logger.debug("Session updated: %s", event)
        model = event.get("model")
        if model is None:
            await self.send_error("Missing required field: model", "invalid_event")
            return
        err = self._check_model(model)
        if err is not None:
            await self.send_error(err.error.message, "model_not_found")
            return
        self._is_model_validated = True
    elif event_type == "input_audio_buffer.append":
        append_event = InputAudioBufferAppend(**event)
        try:
            audio_bytes = base64.b64decode(append_event.audio)
            audio_array = (
                np.frombuffer(audio_bytes, dtype=np.int16).astype(np.float32)
                / 32768.0
            )

            if len(audio_array) / 1024**2 > self._max_audio_filesize_mb:
                raise VLLMValidationError(
                    "Maximum file size exceeded",
                    parameter="audio_filesize_mb",
                    value=len(audio_array) / 1024**2,
                )
            if len(audio_array) == 0:
                raise VLLMValidationError("Can't process empty audio.")

            self.audio_queue.put_nowait(audio_array)

        except Exception as e:
            logger.error("Failed to decode audio: %s", e)
            await self.send_error("Invalid audio data", "invalid_audio")

    elif event_type == "input_audio_buffer.commit":
        if not self._is_model_validated:
            err_msg = (
                "Model not validated. Make sure to validate the"
                " model by sending a session.update event."
            )
            await self.send_error(
                err_msg,
                "model_not_validated",
            )
            return

        commit_event = InputAudioBufferCommit(**event)
        if commit_event.final:
            self.audio_queue.put_nowait(None)
        else:
            await self.start_generation()
    else:
        await self.send_error(f"Unknown event type: {event_type}", "unknown_event")
  1. 方法先取 event["type"]，整个 realtime 输入协议就是靠这个字段分发。
  2. session.update 只做模型校验。没有通过这个事件，后面的 commit 不会启动生成。
  3. input_audio_buffer.append 把客户端发来的 base64 PCM16 解码成 bytes。
  4. np.frombuffer(..., dtype=np.int16).astype(np.float32) / 32768.0 把 PCM16 归一化成模型 realtime buffer 消费的 float32 波形。
  5. 文件大小和空音频校验都发生在入队前；校验失败不会污染 audio_queue。
  6. self.audio_queue.put_nowait(audio_array) 是音频输入流的第一个真实 buffer。append 本身不启动推理，只累积数据。
  7. input_audio_buffer.commit 有两种语义：final=false 启动 generation；final=true 放入 None sentinel，告诉 audio generator 音频结束。
  8. 这里没有 text/video 事件；/v1/realtime 当前这条链路是 audio-first。

上游 vLLM 类 RealtimeConnection，方法 start_generation

async def start_generation(self):
    """Start the transcription generation task."""
    if self.generation_task is not None and not self.generation_task.done():
        logger.warning("Generation already in progress, ignoring commit")
        return

    audio_stream = self.audio_stream_generator()
    input_stream = asyncio.Queue[list[int]]()

    streaming_input_gen = self.serving.transcribe_realtime(
        audio_stream, input_stream
    )

    self.generation_task = asyncio.create_task(
        self._run_generation(streaming_input_gen, input_stream)
    )
  1. 已有 generation task 未结束时，重复 commit 会被忽略，避免同一个 websocket connection 同时跑两条 engine request。
  2. audio_stream_generator() 把 audio_queue 包装成 async generator；它遇到 None sentinel 会停止。
  3. input_stream 是反向上下文通道：模型输出的新 token id 会被放回这个 queue，供下一轮 realtime prompt 使用。
  4. serving.transcribe_realtime(...) 把裸音频流转换成 engine 可消费的 StreamingInput 流。
  5. 真正的 engine 调用不在这里直接 await，而是创建后台 task，由 Omni 子类的 _run_generation 负责消费输出并发 websocket 事件。

上游 vLLM 类 OpenAIServingRealtime，方法 transcribe_realtime

async def transcribe_realtime(
    self,
    audio_stream: AsyncGenerator[np.ndarray, None],
    input_stream: asyncio.Queue[list[int]],
) -> AsyncGenerator[StreamingInput, None]:
    """Transform audio stream into StreamingInput for engine.generate()."""
    model_config = self.model_config
    renderer = self.renderer

    stream_input_iter = cast(
        AsyncGenerator[PromptType, None],
        self.model_cls.buffer_realtime_audio(
            audio_stream, input_stream, model_config
        ),
    )

    async for prompt in stream_input_iter:
        parsed_prompt = parse_model_prompt(model_config, prompt)
        (engine_input,) = await renderer.render_cmpl_async([parsed_prompt])

        yield StreamingInput(prompt=engine_input)
  1. 这个方法不懂 Qwen3-Omni 的切片策略，只负责把模型类产出的 prompt 渲染成 engine input。
  2. self.model_cls.buffer_realtime_audio(...) 是模型定制点；Qwen3-Omni 就是在这里把连续音频切成 segment。
  3. parse_model_prompt 把 TokensPrompt / dict prompt 统一成 renderer 可处理的结构。
  4. renderer.render_cmpl_async 走 chat/completion renderer，输出 vLLM engine input。
  5. yield StreamingInput(prompt=engine_input) 是 engine streaming input 的边界：之后 AsyncOmni 会把第一次 yield 当 add request，后续 yield 当 streaming update。

Qwen3-Omni 类 Qwen3OmniMoeForConditionalGeneration，方法 buffer_realtime_audio

@classmethod
async def buffer_realtime_audio(
    cls,
    audio_stream: AsyncGenerator[np.ndarray, None],
    input_stream: asyncio.Queue[list[int]],
    model_config: ModelConfig,
) -> AsyncGenerator[PromptType, None]:
    processor = cached_processor_from_config(model_config)
    feature_extractor = processor.feature_extractor
    sampling_rate = feature_extractor.sampling_rate
    tokenizer = cached_tokenizer_from_config(model_config)

    segment_duration_s = 5.0
    buffer = Qwen3ASRRealtimeBuffer(
        sampling_rate=sampling_rate,
        segment_duration_s=segment_duration_s,
    )

    audio_placeholder = Qwen3OmniMoeThinkerForConditionalGeneration.get_placeholder_str("audio", 0)
    prompt_template = f"<|im_start|>user\n{audio_placeholder}<|im_end|>\n<|im_start|>assistant\n"

    prompt_token_ids = tokenizer.encode(prompt_template)

    async_chunk = getattr(model_config, "async_chunk", False)

    async for audio_chunk in audio_stream:
        buffer.write_audio(audio_chunk)

        if async_chunk:
            while (segment := buffer.read_audio()) is not None:
                yield TokensPrompt(
                    prompt_token_ids=prompt_token_ids,
                    multi_modal_data={"audio": segment},
                )

    remaining = buffer.flush()
    if remaining is not None and len(remaining) > 0:
        yield TokensPrompt(
            prompt_token_ids=prompt_token_ids,
            multi_modal_data={"audio": remaining},
        )
  1. 先从 model_config 取 HF processor、feature extractor、sampling rate 和 tokenizer。采样率必须来自模型 processor，不能用客户端声称的值。
  2. segment_duration_s = 5.0 定义模型侧低延迟切片大小；客户端 append 的 chunk 可以更小。
  3. Qwen3ASRRealtimeBuffer 负责把多个 websocket audio chunk 拼成稳定的 5 秒左右 segment。
  4. audio_placeholder 生成 Qwen3-Omni Thinker 识别的音频占位符。
  5. prompt_template 是固定 user audio + assistant 开头；每个 segment 复用同一段 token 模板，真正不同的是 multi_modal_data["audio"]。
  6. async_chunk 决定中间 segment 是否可以提前 yield。开启时，buffer 每读出一个完整 segment 就 yield。
  7. yield 的对象是 TokensPrompt：token ids 是固定模板，音频 numpy segment 放在 multi_modal_data。
  8. 音频流结束后调用 flush()。如果还有尾巴音频，不管是否 async_chunk，都要最后 yield 一次，避免丢尾音频。
  9. input_stream 在这个方法当前实现里没有直接读取；它由 realtime connection 传入，保留给需要把上一轮生成 token 作为下一轮上下文的模型实现。

3. engine streaming input：第一次 add request，后面 streaming update

vLLM-Omni 类 AsyncOmni，方法 _add_streaming_input_request

async def _add_streaming_input_request(
    self,
    *,
    request_id: str,
    input_stream: AsyncGenerator[StreamingInput, None],
    sampling_params_list: Sequence[OmniSamplingParams],
    final_stage_id: int,
) -> asyncio.Task:
    """Submit a streaming input generator as incremental stage-0 updates."""
    if not sampling_params_list:
        raise ValueError("sampling_params_list cannot be empty for streaming input")
    stage0_params = sampling_params_list[0]
    self._validate_streaming_input_sampling_params(stage0_params)
    req_state = self.request_states[request_id]
    has_submitted_first_chunk = False

    if not stage0_params.skip_clone:
        stage0_params = stage0_params.clone()
        stage0_params.skip_clone = True

    async def handle_inputs() -> None:
        nonlocal has_submitted_first_chunk
        cancelled = False
        try:
            async for chunk in input_stream:
                chunk_params = getattr(chunk, "sampling_params", None) or stage0_params
                self._validate_streaming_input_sampling_params(chunk_params)
                chunk_sampling_params_list = list(sampling_params_list)
                chunk_sampling_params_list[0] = chunk_params
                chunk_prompt = chunk.prompt
                prompt_text, _, _ = extract_prompt_components(self.model_config, chunk_prompt)

                if not has_submitted_first_chunk:
                    await self.engine.add_request_async(
                        request_id=request_id,
                        prompt=chunk_prompt,
                        prompt_text=prompt_text,
                        sampling_params_list=chunk_sampling_params_list,
                        final_stage_id=final_stage_id,
                        resumable=True,
                    )
                    has_submitted_first_chunk = True
                else:
                    await self.engine.add_streaming_update_async(
                        request_id=request_id,
                        prompt=chunk_prompt,
                        prompt_text=prompt_text,
                        sampling_params_list=chunk_sampling_params_list,
                        final_stage_id=final_stage_id,
                        resumable=True,
                    )
        except (asyncio.CancelledError, GeneratorExit):
            cancelled = True
        except Exception as error:
            await req_state.queue.put(
                ErrorMessage(
                    request_id=request_id,
                    error=str(error),
                )
            )
        finally:
            if not cancelled:
                final_sampling_params_list = list(sampling_params_list)
                final_sampling_params_list[0] = stage0_params
                final_prompt = TokensPrompt(prompt_token_ids=[0])

                if has_submitted_first_chunk:
                    await self.engine.add_streaming_update_async(
                        request_id=request_id,
                        prompt=final_prompt,
                        prompt_text=None,
                        sampling_params_list=final_sampling_params_list,
                        final_stage_id=final_stage_id,
                        resumable=False,
                    )
                else:
                    await self.engine.add_request_async(
                        request_id=request_id,
                        prompt=final_prompt,
                        prompt_text=None,
                        sampling_params_list=final_sampling_params_list,
                        final_stage_id=final_stage_id,
                        resumable=False,
                    )

    input_stream_task = asyncio.create_task(handle_inputs())
    req_state.input_stream_task = input_stream_task
    return input_stream_task
  1. 这个方法只处理 engine 级 streaming input，因此入口参数是 AsyncGenerator[StreamingInput]。
  2. 先校验 sampling params list，并只拿 Stage-0 的参数做 streaming 约束。
  3. has_submitted_first_chunk 是关键状态：它决定当前 chunk 是创建新 request，还是更新已有 request。
  4. 如果 Stage-0 参数还没标记 skip_clone，先 clone 一次并设置 skip，避免每个 segment 都重复深拷贝。
  5. 内部 handle_inputs 作为后台 task 消费 input stream；它不会阻塞外层 generate 的输出消费。
  6. 每个 StreamingInput 可以自带 sampling params；没有就复用 Stage-0 参数。
  7. 第一次 chunk 调 add_request_async(..., resumable=True)，在 scheduler 里创建可续接 request。
  8. 后续 chunk 调 add_streaming_update_async(..., resumable=True)，语义是同一个 request 的下一段输入。
  9. 输入 generator 正常结束时，会发送一个 TokensPrompt([0]) 的 final update，并把 resumable=False。这是整条 streaming request 的终止信号。
  10. 如果输入 task 被取消，不发送 final update，因为上层通常会 abort request。

vLLM-Omni 类 RealtimeConnection，方法 _run_generation

async def _run_generation(
    self,
    streaming_input_gen: AsyncGenerator,
    input_stream: asyncio.Queue[list[int]],
):
    request_id = f"rt-{self.connection_id}-{uuid4()}"
    sent_audio = False
    audio_done_sent = False
    full_text = ""
    prompt_token_ids_len = 0
    completion_tokens_len = 0
    self._realtime_audio_ref = None

    sampling_params_list = list(self.engine.default_sampling_params_list)
    sampling_params_list = coerce_param_message_types(
        sampling_params_list,
        is_streaming=True,
    )

    try:
        result_gen = self.engine.generate(
            prompt=streaming_input_gen,
            request_id=request_id,
            sampling_params_list=sampling_params_list,
        )

        async for output in result_gen:
            stage_id = getattr(output, "stage_id", None)
            if stage_id == 0 and output.outputs:
                first_output = output.outputs[0]
                new_token_ids = list(first_output.token_ids)
                if new_token_ids:
                    input_stream.put_nowait(new_token_ids)

                if output.prompt_token_ids:
                    prompt_token_ids_len = max(
                        prompt_token_ids_len,
                        len(output.prompt_token_ids),
                    )

                delta_text = first_output.text or ""
                full_text += delta_text
                completion_tokens_len += len(new_token_ids)

                if delta_text:
                    await self.send(TranscriptionDelta(delta=delta_text))

            audio_chunks, sample_rate = self._extract_audio_chunks(output)

            for chunk in audio_chunks:
                sent_audio = True
                await self.send_json(
                    {
                        "type": "response.audio.delta",
                        "audio": self._pcm16_b64(chunk),
                        "format": "pcm16",
                        "sample_rate_hz": sample_rate,
                    }
                )

            if not self._is_connected:
                break

        usage = UsageInfo(
            prompt_tokens=prompt_token_ids_len,
            completion_tokens=completion_tokens_len,
            total_tokens=prompt_token_ids_len + completion_tokens_len,
        )
        await self.send(TranscriptionDone(text=full_text, usage=usage))

        if sent_audio:
            await self.send_json({"type": "response.audio.done", "has_audio": True})
            audio_done_sent = True
    except Exception as e:
        logger.exception("Error in generation: %s", e)
        await self.send_error(str(e), "processing_error")
    finally:
        if self._is_connected and not audio_done_sent:
            try:
                await self.send_json({"type": "response.audio.done", "has_audio": sent_audio})
            except Exception:
                logger.exception("Failed to send response.audio.done")
        while not self.audio_queue.empty():
            self.audio_queue.get_nowait()
  1. 方法开头生成 realtime request id，并清空音频去重 ref；每次 websocket generation 都要从空 ref 开始。
  2. coerce_param_message_types(..., is_streaming=True) 把默认输出参数转成适合 streaming 的 delta 形式。
  3. self.engine.generate(prompt=streaming_input_gen, ...) 是关键调用：这里传入的是 streaming generator，不是一次性 prompt。
  4. 循环里先看 stage_id。只有 Stage-0 的文本输出会被当作 transcription delta。
  5. Stage-0 新 token id 会放回 input_stream，保留给后续 streaming prompt 的上下文机制。
  6. delta_text 累计进 full_text，同时以 TranscriptionDelta 发给客户端。
  7. _extract_audio_chunks 从 Omni output 的 multimodal_output 里抽音频，并做累计 waveform 去重。
  8. 每个音频 chunk 被转成 PCM16 base64，通过 response.audio.delta 发出。
  9. engine generator 结束后，先发 TranscriptionDone，再按需发 response.audio.done。
  10. finally 兜底发送 audio done，避免客户端一直等；最后清空音频 queue，防止旧输入污染下一轮。

4. 文本输入与文本流式输出

Qwen3-Omni 的文本输入在当前代码里通常不是 engine streaming input。它要么来自普通 chat request 的 messages，要么来自 video websocket 的 video.query.text。流式主要发生在输出：chat SSE 或 video stream 的 response.text.delta。

vLLM-Omni 类 OmniInputPreprocessor，方法 _process_text

def _process_text(
    self,
    parsed_content: OmniTextPrompt,
    tokenization_kwargs: dict[str, Any] | None = None,
    *,
    mm_uuids: Any | None = None,
) -> OmniTokenInputs | MultiModalInput:
    """Process text prompts with support for mm_processor_kwargs."""
    prompt_text = parsed_content["prompt"]
    mm_processor_kwargs = parsed_content.get("mm_processor_kwargs") or {}
    effective_mm_uuids = mm_uuids or parsed_content.get("multi_modal_uuids")

    inputs: OmniTokenInputs | MultiModalInput
    if multi_modal_data := parsed_content.get("multi_modal_data"):
        inputs = self._process_multimodal(
            prompt_text,
            multi_modal_data,
            mm_processor_kwargs,
            tokenization_kwargs=tokenization_kwargs,
            mm_uuids=effective_mm_uuids,
        )
        prompt_embeds = parsed_content.get("prompt_embeds")
        if prompt_embeds is not None:
            inputs["prompt_embeds"] = prompt_embeds
        additional_information = parsed_content.get("additional_information")
        if additional_information is not None:
            inputs["additional_information"] = additional_information
    elif "mm_processor_kwargs" in parsed_content:
        inputs = self._process_multimodal(
            prompt_text,
            {},
            mm_processor_kwargs,
            tokenization_kwargs=tokenization_kwargs,
            mm_uuids=effective_mm_uuids,
        )
    else:
        prompt_token_ids = self._tokenize_prompt(
            prompt_text,
            tokenization_kwargs=tokenization_kwargs,
        )
        inputs = token_inputs_omni(
            prompt_token_ids,
            prompt_embeds=parsed_content.get("prompt_embeds"),
            additional_information=parsed_content.get("additional_information"),
        )
    prompt_embeds = parsed_content.get("prompt_embeds")
    if prompt_embeds is not None:
        inputs["prompt_embeds"] = prompt_embeds
    additional_information = parsed_content.get("additional_information")
    if additional_information is not None:
        inputs["additional_information"] = additional_information
    if cache_salt := parsed_content.get("cache_salt"):
        inputs["cache_salt"] = cache_salt

    return inputs
  1. 方法入口是已经 parse 好的 text prompt，核心字段是 prompt。
  2. mm_processor_kwargs 即使为空也有意义：有些多模态模型需要通过 processor kwargs 触发 prompt scaffold。
  3. 如果存在 multi_modal_data，文本不会单独 tokenize，而是和 audio/image/video 一起交给 _process_multimodal。
  4. 多模态分支会保留 prompt_embeds 和 additional_information，这对 Omni 多 stage 传递很重要。
  5. 如果没有多模态数据但显式带了 mm_processor_kwargs，仍然走 multimodal processor，方便模型用 processor 默认值构造输入。
  6. 纯文本分支才调用 _tokenize_prompt，然后用 token_inputs_omni 包成 Omni token input。
  7. 方法尾部再次统一补 prompt_embeds、additional_information、cache_salt，保证各分支输出字段一致。
  8. 所以普通文本输入的本质是“一次 request 的 token prompt”，不是像 audio realtime 那样多次 yield StreamingInput。

class StreamingVideoSessionConfig(BaseModel):
    """Configuration sent as the first WebSocket message."""

    model: str | None = None
    modalities: list[str] = Field(
        default_factory=lambda: ["text", "audio"],
        description="Output modalities: 'text', 'audio', or both.",
    )
    num_frames: int = Field(
        default=4,

        ge=1,

    )
    system_prompt: str | None = Field(

        description="Custom system prompt.",
    )

vLLM-Omni 类 OmniStreamingVideoHandler，方法 _build_messages

def _build_messages(
    self,
    config: StreamingVideoSessionConfig,
    frame_buffer: list[str],
    audio_buffer: bytearray,
    message_history: list[dict[str, Any]],
    query_text: str,
    prewarmed_frames: dict[str, tuple[Any, str]],
) -> tuple[list[dict[str, Any]], dict[str, Any]]:
    """Build OpenAI-style messages list and the current user message."""
    n_buf = len(frame_buffer)
    if n_buf <= config.num_frames:
        frames = list(frame_buffer)
    else:
        stride = max(1, n_buf // config.num_frames)
        idx = [i * stride for i in range(config.num_frames - 1)] + [n_buf - 1]
        frames = [frame_buffer[i] for i in idx]

    prewarmed = prewarmed_frames or {}
    user_content: list[dict] = []
    for frame_b64 in frames:
        cached = prewarmed.get(frame_b64)
        if cached is _BAD_FRAME:
            continue
        if cached is not None:
            pil, pil_uuid = cached
            user_content.append(
                {
                    "type": "image_pil",
                    "image_pil": pil,
                    "uuid": pil_uuid,
                }
            )
        else:
            user_content.append(
                {
                    "type": "image_url",
                    "image_url": {"url": f"data:image/jpeg;base64,{frame_b64}"},
                }
            )

    if len(audio_buffer) > 0:
        wav_b64 = self._pcm_to_wav_b64(bytes(audio_buffer))
        user_content.append(
            {
                "type": "input_audio",
                "input_audio": {
                    "data": wav_b64,
                    "format": "wav",
                },
            }
        )

    if query_text:
        user_content.append({"type": "text", "text": query_text})

    user_message: dict[str, Any] = {"role": "user", "content": user_content}

    messages: list[dict[str, Any]] = []
    if config.system_prompt:
        messages.append({"role": "system", "content": config.system_prompt})

    recent_history = message_history[-2:] if len(message_history) > 2 else message_history
    for hist_msg in recent_history:
        messages.append(self._text_only_message(hist_msg))

    messages.append(user_message)

    return messages, user_message
  1. 先按 config.num_frames 对 session frame buffer 做采样；不是每一帧都进模型。
  2. buffer 较长时，用 stride 采样，并强制保留最新一帧。
  3. 每个采样帧优先使用预热好的 PIL 对象和 uuid，这能减少 query 时解码成本，也方便多模态 cache 命中。
  4. 预热失败的帧会跳过，不会进入 prompt。
  5. 没有预热对象时退回 image_url data URI，仍然是 OpenAI-style content block。
  6. 如果当前 query 带音频 buffer，就把 PCM16 包成 WAV base64，作为 input_audio content block。
  7. query 文本作为 {"type": "text"} 附在最后，所以模型看到的是“帧 + 可选音频 + 文本问题”。
  8. 历史消息只保留最近一轮，并且通过 _text_only_message 去掉历史里的图片/音频，避免 prompt 太长和重复多模态编码。
  9. 返回的 messages 会被包装成 ChatCompletionRequest，这一步之后就走普通 chat/render/preprocess 管线。

Qwen3-Omni 类 Qwen3OmniMoeThinkerMultiModalProcessor，方法 _call_hf_processor

def _call_hf_processor(
    self,
    prompt: str,
    mm_data: Mapping[str, object],
    mm_kwargs: Mapping[str, object],
    tok_kwargs: Mapping[str, object],
) -> BatchFeature:
    mm_data = dict(mm_data)
    audios = mm_data.pop("audios", [])

    def pad_to_hop_length(x: np.ndarray, hop_length: int) -> np.ndarray:
        length = x.shape[-1]
        if length % hop_length != 0:
            pad_length = hop_length - (length % hop_length)
            x = np.pad(x, (0, pad_length), mode="constant", constant_values=0)
        return x

    feature_extractor = self.info.get_feature_extractor(**mm_kwargs)
    hop_length = feature_extractor.hop_length
    if audios:
        mm_data["audio"] = [
            pad_to_hop_length(audio, hop_length)
            if isinstance(audio, np.ndarray)
            else (pad_to_hop_length(audio[0], hop_length), audio[1])
            for audio in audios
        ]

        mm_kwargs = dict(mm_kwargs)
        tok_kwargs = dict(tok_kwargs)
        mm_kwargs["audio_kwargs"] = dict(mm_kwargs.get("audio_kwargs") or {})
        mm_kwargs["text_kwargs"] = dict(mm_kwargs.get("text_kwargs") or {})
        if Version(TRANSFORMERS_VERSION) < Version("4.58.0"):
            audio_sample_rate = mm_kwargs.pop("audio_sample_rate", None)

            mm_kwargs["audio_kwargs"].setdefault("truncation", mm_kwargs.pop("truncation", False))
            mm_kwargs["text_kwargs"].setdefault("truncation", tok_kwargs.pop("truncation", False))

            if audio_sample_rate is not None:
                expected_sr = feature_extractor.sampling_rate
                if audio_sample_rate != expected_sr:
                    logger.warning(
                        "[%s] audio_sample_rate mismatch: user provided %dHz "
                        "but model expects %dHz. Ignoring user value. "
                        "vLLM's audio loader already resampled to %dHz.",
                        self.__class__.__name__,
                        audio_sample_rate,
                        expected_sr,
                        expected_sr,
                    )
                else:
                    mm_kwargs["audio_kwargs"]["audio_sample_rate"] = audio_sample_rate

    hf_inputs = super()._call_hf_processor(
        prompt=prompt,
        mm_data=mm_data,
        mm_kwargs=mm_kwargs,
        tok_kwargs=tok_kwargs,
    )

    if (
        "audio_feature_lengths" in hf_inputs
        and "feature_attention_mask" in hf_inputs
        and (audios := mm_data.get("audio", []))
    ):
        audio_num_frames = []
        for _, audio in enumerate(audios):
            audio_length = len(audio[0]) if isinstance(audio, tuple) else len(audio)
            num_frame = (
                (audio_length // hop_length) if audio_length % hop_length == 0 else (audio_length // hop_length - 1)
            )
            if mm_kwargs.get("truncation", False):
                num_frame = min(num_frame, feature_extractor.n_samples // hop_length)
            audio_num_frames.append(num_frame)
        hf_inputs["feature_attention_mask"] = [torch.ones(num_frame) for num_frame in audio_num_frames]
        hf_inputs["audio_feature_lengths"] = torch.tensor(audio_num_frames)
    return hf_inputs
  1. 方法收到 renderer/preprocessor 汇总后的 prompt、mm_data、mm_kwargs、tokenizer kwargs。
  2. 先把 audios 从 mm_data 里弹出，因为 Qwen3-Omni HF processor 期望字段名是 audio。
  3. pad_to_hop_length 把音频长度补齐到 feature extractor hop length 的整数倍，保证音频 token/frame 长度可预测。
  4. 如果有音频，统一转成 mm_data["audio"]，并保留 tuple 形式音频的采样率/元信息。
  5. 旧 transformers 版本下，把 truncation 拆到 audio_kwargs 和 text_kwargs，避免 HF processor 参数冲突。
  6. 如果用户传了 audio sample rate，只在它等于模型 expected sample rate 时向下传；否则忽略并 warning，因为 vLLM loader 已经做过重采样。
  7. 随后调用父类 processor 处理 image/video/audio/text，生成 HF BatchFeature。
  8. 最后修正音频的 feature_attention_mask 和 audio_feature_lengths，让模型只注意真实音频帧，不把 padding 当有效音频。

Qwen3-Omni 类 Qwen3OmniMoeThinkerMultiModalProcessor，方法 _maybe_apply_prompt_updates

def _maybe_apply_prompt_updates(
    self,
    mm_items: MultiModalDataItems,
    prompt_ids: list[int],
    mm_kwargs: MultiModalKwargsItems,
    mm_prompt_updates: MultiModalPromptUpdates,
    is_update_applied: bool,
) -> tuple[list[int], str, Mapping[str, list[PlaceholderFeaturesInfo]]]:
    """
    Qwen3-Omni reimplements this function to handle `use_audio_in_video`.
    """
    mm_item_counts = mm_items.get_all_counts()
    self._validate_mm_kwargs(mm_kwargs, mm_item_counts)

    use_audio_in_video = False
    if "video" in mm_kwargs:
        non_none_items = [item for item in mm_kwargs["video"] if item is not None]
        if non_none_items:
            use_audio_in_video = any(item["use_audio_in_video"].data for item in non_none_items)
        elif "audio" in mm_prompt_updates:
            tokenizer = self.info.get_tokenizer()
            audio_pad_id = tokenizer.convert_tokens_to_ids("<|audio_pad|>")
            use_audio_in_video = audio_pad_id not in prompt_ids
        if any(item is None for item in mm_kwargs["video"]):
            video_token_id = self.info.get_hf_config().video_token_id
            audio_token_id = self.info.get_hf_config().audio_token_id
            video_audio_item_num = sum(id in (video_token_id, audio_token_id) for id in prompt_ids)
            audio_updates_num = len(mm_prompt_updates.get("audio", []))
            video_updates_num = len(mm_prompt_updates.get("video", []))
            if video_audio_item_num != video_updates_num + audio_updates_num:
                use_audio_in_video = True

    if is_update_applied:
        mm_placeholders = self._find_mm_placeholders(
            prompt_ids,
            mm_prompt_updates,
        )
        self._validate_mm_placeholders(
            mm_placeholders,
            mm_item_counts,
        )
    else:
        if use_audio_in_video and "audio" in mm_prompt_updates:
            filtered_updates = {k: v for k, v in mm_prompt_updates.items() if k != "audio"}
            prompt_ids, mm_placeholders = self._apply_prompt_updates(
                prompt_ids,
                filtered_updates,
            )
            mm_placeholders = self._derive_audio_from_video_placeholders(mm_placeholders, mm_prompt_updates)
        else:
            prompt_ids, mm_placeholders = self._apply_prompt_updates(
                prompt_ids,
                mm_prompt_updates,
            )

        self._validate_mm_placeholders(
            mm_placeholders,
            mm_item_counts,
        )

    return prompt_ids, mm_placeholders
  1. 这个方法决定多模态 placeholder token 怎么写进 prompt ids，并且是 Qwen3-Omni 处理 video+audio 交织的关键。
  2. 先统计每种模态 item 数量，然后校验 mm_kwargs 与 item 数是否匹配。
  3. 默认 use_audio_in_video=False。只有看到 video kwargs 或 cached video 特征时，才判断是否需要把 audio 嵌到 video token 序列里。
  4. 正常非缓存 video item 可以直接读取 item["use_audio_in_video"].data。
  5. 如果 video item 都来自 cache，则通过 prompt 里有没有 <|audio_pad|> 来推断 audio 是否已经嵌在 video tokens 中。
  6. 缓存场景还会比较 prompt 里的 video/audio token 数量与 prompt updates 数量；不一致时倾向认为是 audio-in-video。
  7. 如果 prompt updates 已经应用过，只查找并校验 placeholders。
  8. 如果还没应用，并且 use_audio_in_video 为真，就过滤掉独立 audio update，只应用 video update，再用 _derive_audio_from_video_placeholders 推导 audio placeholder。
  9. 否则走普通路径，把 audio/image/video updates 都应用到 prompt。
  10. 返回的是更新后的 prompt_ids 和各模态 placeholder 位置信息，后续 embedding merge 依赖它们。

def embed_multimodal(self, **kwargs: object) -> MultiModalEmbeddings | None:
    mm_input_by_modality = self._parse_and_validate_multimodal_inputs(**kwargs)
    if not mm_input_by_modality:
        return []

    multimodal_embeddings: tuple[torch.Tensor, ...] = ()

    for modality in mm_input_by_modality:
        multimodal_input = mm_input_by_modality[modality]
        if modality == "image":
            image_embeddings = self._process_image_input(multimodal_input)
            multimodal_embeddings += tuple(image_embeddings)
        if modality == "video":
            video_embeddings = self._process_video_input(multimodal_input)
            multimodal_embeddings += tuple(video_embeddings)
        if modality == "audio":
            audio_embeddings = self._process_audio_input(multimodal_input)
            multimodal_embeddings += tuple(audio_embeddings)
    return multimodal_embeddings
  1. 先从 kwargs 里解析并校验多模态输入，得到按模态分组的结构。
  2. 没有多模态输入时返回空 list，后面走纯文本 embedding。
  3. multimodal_embeddings 是 tuple，顺序很重要，因为后面要按 prompt placeholder 顺序 merge。
  4. image 走视觉 image processor；video 走视频 processor；audio 走音频 encoder。
  5. 每个模态的结果都追加到同一个 tuple，最终交给 embed_input_ids 合并到文本 token embedding 里。

  1. 先把所有 token ids 都用语言模型 embedding 层转成文本 embedding。
  2. 没有多模态 embedding 时直接返回文本 embedding。
  3. is_multimodal 是 placeholder mask，先搬到和 input ids 相同设备。
  4. 通过 Qwen3 config 里的 video/audio token id，分别找出 video placeholder 和 audio placeholder。
  5. check_interleaved_audio_video 判断 prompt 里 video/audio token 是否交织排列。
  6. 如果是 audio-in-video 交织模式，用 merge_interleaved_embeddings 按 video/audio mask 精确合并。
  7. 否则走父类标准合并逻辑，把多模态 embeddings 按普通 placeholder 顺序塞回文本 embedding 序列。

7. 读码结论

  • /v1/realtime 的音频输入是真正的 engine streaming input：append 入队，commit 启动，Qwen3 按 5 秒 segment yield StreamingInput。
  • AsyncOmni._add_streaming_input_request 是 add request 与 streaming update 的分界点：第一片 resumable=True add，后续 resumable=True update，结束 resumable=False update。
  • 文本输入本身通常是一次 prompt 的一部分；文本“流式”主要是输出侧 response.text.delta 或 chat SSE delta。
  • /v1/video/chat/stream 是 websocket 层流式收帧：每次 video.query 把当前帧 buffer、音频 buffer、query text 快照成一次 ChatCompletionRequest。
  • Qwen3-Omni Thinker 的 multimodal processor 负责把 audio/image/video placeholder、feature lengths、interleaved audio-in-video 关系处理正确。
