2. 数学原理

2.1 Router

输入: X in R[T, H] W_gate in R[E, H] router logits: Z = X @ W_gate.T # [T, E] DeepSeek-V3 scoring: S = sigmoid(Z) # [T, E] S_select = S + correction_bias # [T, E], 仅用于选择，不用于最终权重 分组: E = 256, G = 8, group_size = 32 对每个 token，每组取 top-2 分数求和作为 group_score group_idx = topk(group_score, G_top=4) 在这 4 个 group 覆盖的 experts 中取 top-K=8 experts 路由权重: W = gather(S, topk_ids) # [T, K] if norm_topk_prob: W = W / sum(W, axis=-1, keepdim=True) W = W * routed_scaling_factor # DeepSeek-V3 为 2.5，vLLM 可能等价地延后乘到 output

2.2 Expert FFN

第 e 个 routed expert: W1_e in R[I, H] # gate projection W3_e in R[I, H] # up projection W2_e in R[H, I] # down projection expert_e(x) = W2_e @ ( SiLU(W1_e @ x) * (W3_e @ x) ) # R[H] vLLM 存储: w13_weight[e] = concat(W1_e, W3_e) # [2I, H] w2_weight[e] # [H, I]

2.3 MoE 输出

对第 t 个 token: y_routed[t] = sum_{j=1..K} W[t, j] * expert_{topk_ids[t, j]}(X[t]) shared expert: y_shared[t] = shared_mlp(X[t]) # DeepSeek-V3 n_shared_experts=1 最终: Y[t] = y_routed[t] + y_shared[t]

def forward(self, hidden_states: torch.Tensor) -> torch.Tensor:
    num_tokens, hidden_dim = hidden_states.shape
    hidden_states = hidden_states.view(-1, hidden_dim)

    if self.is_sequence_parallel:
        hidden_states = sequence_parallel_chunk(hidden_states)

    if self.experts.is_internal_router:
        final_hidden_states = self.experts(
            hidden_states=hidden_states, router_logits=hidden_states
        )
    else:
        router_logits, _ = self.gate(hidden_states)
        final_hidden_states = self.experts(
            hidden_states=hidden_states, router_logits=router_logits
        )

            final_hidden_states, 0
        )
        final_hidden_states = final_hidden_states[:num_tokens]

6. Grouped Top-K 路由代码

DeepSeek-V3 的路由在 vLLM 中走 GroupedTopk。关键点：有 e_score_correction_bias 时，biased score 用于选择 expert；最终路由权重仍从原始 sigmoid score 中 gather。

# vllm/vllm/model_executor/layers/fused_moe/router/grouped_topk_router.py:111-162
if scoring_func == "softmax":
    scores = torch.softmax(gating_output, dim=-1)
elif scoring_func == "sigmoid":
    scores = gating_output.sigmoid()

num_token = scores.size(0)
if e_score_correction_bias is not None:
    original_scores = scores
    scores = scores + e_score_correction_bias.unsqueeze(0)
    group_scores = (
        scores.view(num_token, num_expert_group, -1).topk(2, dim=-1)[0].sum(dim=-1)
    )
else:
    group_scores = (
        scores.view(num_token, num_expert_group, -1).max(dim=-1).values
    )

group_idx = torch.topk(group_scores, k=topk_group, dim=-1, sorted=use_sorted)[1]
group_mask = torch.zeros_like(group_scores)
group_mask.scatter_(1, group_idx, 1)
score_mask = (
    group_mask.unsqueeze(-1)
    .expand(num_token, num_expert_group, scores.size(-1) // num_expert_group)
    .reshape(num_token, -1)
)
tmp_scores = scores.masked_fill(~score_mask.bool(), float("-inf"))

if e_score_correction_bias is not None:
    topk_ids = torch.topk(tmp_scores, k=topk, dim=-1, sorted=use_sorted)[1]
    topk_weights = original_scores.gather(1, topk_ids)
else:
    topk_weights, topk_ids = torch.topk(tmp_scores, k=topk, dim=-1, sorted=use_sorted)

if renormalize:
    topk_weights = topk_weights / topk_weights.sum(dim=-1, keepdim=True)

if routed_scaling_factor != 1.0:
    topk_weights = topk_weights * routed_scaling_factor
return topk_weights.to(torch.float32), topk_ids.to(torch.int32)

7. Kernel 数据流与 Shape

Runner 的主线是：可选 input transform，gate，dispatch，router select，quant/expert kernel，combine，shared+routed 相加。

# vllm/vllm/model_executor/layers/fused_moe/runner/moe_runner.py:744-801
def _forward_impl(
    self,
    layer: torch.nn.Module,
    hidden_states: torch.Tensor,
    router_logits: torch.Tensor,
    shared_experts_input: torch.Tensor | None,
    input_ids: torch.Tensor | None = None,
) -> torch.Tensor | tuple[torch.Tensor, torch.Tensor]:
    layer.ensure_moe_quant_config_init()
    self._maybe_sync_shared_experts_stream(shared_experts_input)

    if self.gate is not None:
        if self._fse_fuse_gate:
            self._maybe_fuse_gate_weights()
            router_logits = F.linear(hidden_states, self._combined_gate_weight)
        else:
            router_logits, _ = self.gate(hidden_states)

    with self._sequence_parallel_context():
        hidden_states, router_logits = self._maybe_dispatch(
            layer,
            hidden_states,
            router_logits,
        )

        shared_output, hidden_states = self._apply_quant_method(
            layer=layer,
            hidden_states=hidden_states,
            router_logits=router_logits,
            shared_experts_input=shared_experts_input,
            input_ids=input_ids,
        )

        return self._maybe_combine(shared_output, hidden_states)

8. EP 怎么实现

8.1 进程组：EP = TP × DP × PCP

# vllm/vllm/distributed/device_communicators/all2all.py:50-137
def dispatch(
    self,
    hidden_states: torch.Tensor,
    topk_weights: torch.Tensor,
    topk_ids: torch.Tensor,
    is_sequence_parallel: bool = False,
    extra_tensors: list[torch.Tensor] | None = None,
):
    dp_metadata = get_forward_context().dp_metadata
    sizes = dp_metadata.get_chunk_sizes_across_dp_rank()
    dist_group = get_ep_group() if is_sequence_parallel else get_dp_group()

    tensors_to_gather = [hidden_states, topk_weights, topk_ids]
    gathered_tensors = dist_group.all_gatherv(
        tensors_to_gather,
        dim=0,
        sizes=sizes,
    )
    return gathered_tensors[0], gathered_tensors[1], gathered_tensors[2]

def combine(self, hidden_states: torch.Tensor, is_sequence_parallel: bool = False):
    dp_metadata = get_forward_context().dp_metadata
    sizes = dp_metadata.get_chunk_sizes_across_dp_rank()
    dist_group = get_ep_group() if is_sequence_parallel else get_dp_group()
    return dist_group.reduce_scatterv(hidden_states, dim=0, sizes=sizes)

9. 大 EP 是什么

大 EP 可以理解为：把 EP_SIZE 扩到很多 GPU，通常用 TP=1, DP=N, --enable-expert-parallel，让 attention 在 DP rank 间复制或小规模 TP，而 MoE experts 横跨所有 DP×TP ranks 分片。

普通 TP MoE

  • 每个 expert 的矩阵被 TP 切开。
  • 所有 rank 都保留所有 experts 的一片。
  • MoE 权重 IO 和显存不随 expert 稀疏性充分下降。

大 EP MoE

  • 每个 rank 持有一部分完整 experts。
  • EP_SIZE 越大，每 rank expert 越少。
  • 代价是 MoE 层每步需要跨 rank token dispatch/combine。
配置EP_SIZE每 rank routed experts适用
TP=8, DP=1, EP on832单节点，attention TP=8，MoE EP=8
TP=1, DP=8, EP on832单节点大 EP，attention 复制，MoE 分片
TP=1, DP=16, EP on1616两节点或更多，常配 DeepEP low_latency/high_throughput
TP=1, DP=32, EP on + EPLB redundant=3232(256+32)/32=9大规模服务，热门 expert 可复制

vLLM 文档也把 EP 描述为通常与 DP 结合，EP size 自动等于 TP_SIZE × DP_SIZE。多节点时建议配 --all2all-backend deepep_high_throughput 用于 prefill，deepep_low_latency 用于 decode。

10. EPLB 与冗余专家

10.1 逻辑专家、物理专家、冗余专家

vLLM 的 EPLB 术语非常直接：logical expert 是模型原始 expert；redundant expert 是热门 logical expert 的额外副本；physical expert 是实际部署在某个设备上的 expert slot。

# vllm/vllm/distributed/eplb/eplb_state.py:8-26
- Logical Expert: An expert that is part of the model's logical structure.
- Redundant Expert: additional copies of popular logical experts.
- Physical Expert: an expert instantiated on a specific device.

For example: DeepSeek-R1 has 256 logical experts.
If we add 32 redundant experts, DeepSeek-R1 will have
256 + 32 = 288 physical experts in total.
With 32 EP ranks, each GPU holds 288 / 32 = 9 local physical experts.

初始化时，先把 physical_to_logical_map 设为 [0,1,...,255,0,1,...]，再反推出 logical_to_physical_map 和 logical_replica_count。每层都有自己的 map。

10.4 重排策略：复制热门 expert，再把 physical experts 打包到 GPU

# vllm/vllm/distributed/eplb/policy/default.py:76-101
def replicate_experts(cls, weight: np.ndarray, num_phy: int):
    n, num_log = weight.shape
    num_redundant = num_phy - num_log
    phy2log = np.tile(np.arange(num_phy, dtype=np.int64), (n, 1))
    logcnt = np.ones((n, num_log), dtype=np.int64)
    arangen = np.arange(n, dtype=np.int64)
    for i in range(num_log, num_phy):
        redundant_indices = np.argmax(weight / logcnt, axis=-1)
        phy2log[:, i] = redundant_indices
        logcnt[arangen, redundant_indices] += 1
    return phy2log, logcnt

直觉：负载越高的 logical expert 越可能被复制。复制后，一个热门 logical expert 的 token 会被 hash 到多个 physical replicas，降低单 rank 峰值。
